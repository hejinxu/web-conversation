## 主题系统架构概述

当前主题系统由以下核心组件组成：

```
config/theme.ts              # 主题配置定义
hooks/use-theme.ts           # 主题状态管理 Hook（包含 body 样式直接设置）
app/components/theme-provider/  # 主题 Context 提供器
app/components/theme-toggle-button/  # 主题切换按钮
app/styles/globals.css       # 全局样式（包含所有主题的 CSS 覆盖规则）
tailwind.config.js           # Tailwind 配置
next.config.js               # Next.js 配置（如 devIndicators）
```

### 主题实现原理

由于项目中大量组件直接使用 Tailwind 的 `dark:` 前缀而非 CSS 变量，Tailwind 的主题前缀（如 `tech-blue:bg-blue-900`）无法自动生效。因此采用以下策略：

1. **Hook 层**：通过 JavaScript 直接设置 `body.style.backgroundColor` 和 `body.style.color`
2. **CSS 层**：在 `globals.css` 中使用高优先级选择器覆盖组件的 Tailwind 样式

## 步骤 1：更新主题配置

首先，在 `config/theme.ts` 中添加新主题：

```typescript
// config/theme.ts

export type ThemeMode = 'light' | 'dark' | 'system' | 'tech-blue'

export const THEME_MODES: Record<string, ThemeMode> = {
  LIGHT: 'light',
  DARK: 'dark',
  SYSTEM: 'system',
  TECH_BLUE: 'tech-blue',  // 新增
}

export const THEME_STORAGE_KEY = 'theme-mode'

export const DEFAULT_THEME: ThemeMode = (process.env.NEXT_PUBLIC_DEFAULT_THEME as ThemeMode) || THEME_MODES.LIGHT

export function isValidThemeMode(mode: string): mode is ThemeMode {
  return Object.values(THEME_MODES).includes(mode as ThemeMode)
}
```

## 步骤 2：扩展主题管理 Hook

在 `hooks/use-theme.ts` 中需要做两件事：

1. 添加新主题类型支持
2. 添加直接设置 body 样式的函数（用于覆盖 Tailwind 预置样式）

```typescript
// hooks/use-theme.ts

export interface UseThemeReturn {
  theme: ThemeMode
  resolvedTheme: 'light' | 'dark' | 'tech-blue'  // 添加新主题类型
  setTheme: (theme: ThemeMode) => void
  toggleTheme: () => void
}

// 添加直接设置 body 样式的函数
function applyBodyStyles(resolvedTheme: 'light' | 'dark' | 'tech-blue') {
  const body = document.body
  if (!body) return

  switch (resolvedTheme) {
    case 'dark':
      body.style.backgroundColor = '#111928'
      body.style.color = '#F9FAFB'
      break
    case 'tech-blue':  // 新主题
      body.style.backgroundColor = 'rgba(36, 62, 92, 0.85)'
      body.style.color = '#E0F2FF'
      break
    default:
      body.style.backgroundColor = '#FFFFFF'
      body.style.color = '#111928'
  }
}

// 在 applyTheme 函数中调用
useEffect(() => {
  const applyTheme = () => {
    // ... 现有逻辑
    root.classList.remove('light', 'dark', 'tech-blue')
    root.classList.add(newResolvedTheme)
    
    // 添加这行
    applyBodyStyles(newResolvedTheme)
  }
  // ...
}, [theme])

// 更新 toggleTheme 函数
const toggleTheme = () => {
  const nextTheme = theme === THEME_MODES.LIGHT
    ? THEME_MODES.DARK
    : theme === THEME_MODES.DARK
      ? THEME_MODES.TECH_BLUE  // 新增
      : theme === THEME_MODES.TECH_BLUE
        ? THEME_MODES.SYSTEM
        : THEME_MODES.LIGHT
  setTheme(nextTheme)
}
```

## 步骤 4：添加主题切换组件选项

在 `app/components/theme-toggle-button/index.tsx` 中添加新主题选项：

```tsx
// app/components/theme-toggle-button/index.tsx

import { THEME_MODES } from '@/config/theme'
import { MoonIcon, SunIcon, ComputerDesktopIcon, SparklesIcon } from '@heroicons/react/24/outline'

// 添加新主题图标和选项
const getThemeIcon = () => {
  if (theme === THEME_MODES.LIGHT) {
    return <SunIcon className="h-4 w-4" />
  }
  if (theme === THEME_MODES.DARK) {
    return <MoonIcon className="h-4 w-4" />
  }
  if (theme === THEME_MODES.TECH_BLUE) {  // 新增
    return <SparklesIcon className="h-4 w-4" />
  }
  return <ComputerDesktopIcon className="h-4 w-4" />
}

// 在下拉菜单中添加选项
<button
  onClick={() => handleDropdownItemClick(THEME_MODES.TECH_BLUE)}
  className={getButtonClassName(THEME_MODES.TECH_BLUE)}
>
  <SparklesIcon className="h-4 w-4" />
  科技蓝
</button>
```

由于组件中大量使用 `dark:` 前缀，可能需要在 `globals.css` 中为每个主题添加完整的样式覆盖。以下是常见需要覆盖的组件：

位置：`app/components/sidebar/index.tsx`

## 测试和验证

### 1. 测试清单

- [ ] 在 `.env.local` 中设置默认主题为新主题，验证页面初始化正确加载
- [ ] 测试主题切换功能
- [ ] 测试 localStorage 保存功能
- [ ] 深色/浅色模式切换
- [ ] 跟随系统功能
- [ ] 新主题完整显示（主体背景、卡片、文字颜色、选中状态、hover 状态）
- [ ] 特殊组件（消息气泡、输入框、按钮等）的显示效果

### 2. 常见问题排查

**问题：页面没有正确应用主题颜色？**

**解决方案：**
1. 检查 `hooks/use-theme.ts` 中是否正确添加了 `applyBodyStyles` 函数调用
2. 检查 `globals.css` 中是否使用了足够高的优先级（添加 `!important`）
3. 检查选择器是否正确（使用 `html.tech-blue` 或 `.tech-blue`）
4. 清除浏览器缓存和 localStorage：`localStorage.removeItem('theme-mode')`
5. 重启开发服务器并清除 `.next` 缓存

**问题：主题切换不生效？**

**解决方案：**
1. 检查 `useTheme` Hook 中的主题切换逻辑
2. 检查 ThemeProvider 是否正确包裹了应用
3. 检查 `config/theme.ts` 中的 `isValidThemeMode` 函数是否包含新主题

**问题：hover 状态不生效？**

**解决方案：**
使用属性选择器覆盖所有 hover 类：
```css
.tech-blue [class*="hover:bg-"]:hover {
  background-color: #2A4D6E !important;
}
```

**问题：深色模式也有类似问题？**

**解决方案：**
同样需要在 `globals.css` 中添加深色模式的样式覆盖：
```css
/* Dark theme - Session list selected state */
.dark .dark\:bg-primary-900\/30,
.dark .bg-primary-50 {
  background-color: rgba(28, 100, 242, 0.2) !important;
}

/* Dark theme - hover state */
.dark [class*="hover:bg-"]:hover {
  background-color: #374151 !important;
}
```

## 最佳实践

1. **保持一致性**：为每个主题定义一套完整的颜色系统
2. **透明度运用**：科技感主题可适当使用透明度（如 `rgba(36, 62, 92, 0.85)`）
3. **可访问性**：确保新主题满足 WCAG 2.1 对比度要求
4. **测试**：在不同设备和浏览器上测试新主题
5. **文档更新**：添加新主题后及时更新本文档
